## 三、架构改进详情

### 3.1 后端领域模块化（阶段 4）

**重构前**（技术分层混杂）：
```
app/
├── models/        (17 个扁平模型文件)
├── schemas/       (扁平 schema)
├── services/      (扁平 service)
├── routers/       (扁平 router)
└── main.py (679 行，含异常 handler + 健康检查 + 种子数据)
```

**重构后**（5 领域闭环）：
```
app/
├── domains/
│   ├── learner/   {models 254, schemas 239, service 702, router 370}
│   ├── knowledge/ {models 109, schemas 182, service 886, router 416}
│   ├── resource/  {models 346, service 347}
│   ├── agent/     {models 99, schemas 118, router 719}
│   └── training/  {models 74, schemas 113, service 282, router 131}
├── routers/       (core/auth/privacy 跨域保留)
├── services/     (tutoring/report/privacy/common/archive 跨域保留)
├── exception_handlers.py (181 行，9 类异常 + 业务文案)
├── health.py      (197 行，健康检查/指标/信息)
├── seed_data.py   (87 行，3 个种子函数)
└── main.py        (115 行，纯装配)
```

**迁移策略**：增量迁移 + `app/models/__init__.py` re-export 兼容，URL 零变更，每域独立验证。

### 3.2 前端 store 分域（阶段 3）

**重构前**：单一 `store/index.ts` 聚合，全组件订阅全 store。
**重构后**：
```
src/store/
├── authStore.ts       (认证 + 用户)
├── learnerStore.ts    (学习者列表/当前)
├── knowledgeStore.ts  (知识库)
├── agentStore.ts      (Agent 任务/状态)
├── resourceStore.ts   (资源生成)
└── index.ts           (兼容 re-export，旧 import 不破)
```
配合 `useShallow` selector memo，重渲染数下降。

### 3.3 main.py 拆分（阶段 4 T4.6）

| 抽取模块 | 行数 | 职责 |
|---|---|---|
| `exception_handlers.py` | 181 | 9 类异常 + 业务文案 + 结构化日志 |
| `health.py` | 197 | 健康检查/就绪/指标/系统信息（11 端点） |
| `seed_data.py` | 87 | 默认管理员 + 培训 + 学习者种子 |
| `main.py`（剩余） | 115 | lifespan + 中间件 + 路由注册 |

### 3.4 E2E + 可观测性（阶段 5）

- **Playwright**：`playwright.config.ts` + 6 spec + auth helper + CSV fixture，21 用例覆盖全链路。
- **Sentry**：`src/lib/sentry.ts` 可选集成，DSN 未配则 no-op；接入 ErrorBoundary、request 5xx、authStore 用户上下文。
- **后端异常**：`_build_error_response` 统一构造，日志带 `request_id/method/path/error_type` 标签，IntegrityError 细分 unique/FK/not-null。
