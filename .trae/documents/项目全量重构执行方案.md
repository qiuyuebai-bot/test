# 项目全量重构执行方案

> 编制日期：2026-07-08 · 待批准后实施 · 范围：代码与测试侧（部署侧 K8s 已完成，不动）

## Context（背景与动因）

本项目（领域知识个性化生成与多智能体协同决策系统）已通过多轮优化处于"运行稳定、可演示"状态，K8s 部署侧 9 阶段 57 任务全部完成（Helm Chart + 监控告警 + 蓝绿/金丝雀 + 备份恢复 + GitOps）。

**但代码与测试侧存在显著短板**：`code-quality-report.md` 识别出 81 项问题（6 P0 安全 + 15 P1 功能 + 30 P2 性能 + 30 P3 规范），测试覆盖率未量化，`main.py` 600 行、`orchestrator.py` 1190 行、`knowledge_service.py` 919 行等大文件职责混杂。部署侧已达生产级，代码侧成为最大短板，需通过本次重构实现"代码-部署"双侧均衡。

**本次核查的重要修正**：既有 `docs/REFACTORING_PLAN.md` v1.0 中"前端无路由懒加载"已过时——`src/App.tsx` L9-21 已用 `React.lazy + Suspense` 实现全部 13 页面懒加载；部分 P0/P1（P0-4 JWT 权限、P1-1 Chroma KeyError、P0-3 日志泄露）已修复；`agent.py` 已见 SSE ticket 机制与 `_mark_task_failed`。方案据此缩减前端范围、上调代码 Bug 修复优先级。

### 1. 领域模块化增量迁移（以 learner 域为例）

```python
if settings.ENABLE_DOMAIN_ROUTER:  # config.py 新增，默认 False
    from app.domains.learner.router import router as learner_router
else:
    from app.routers.learner import router as learner_router
app.include_router(learner_router, prefix=settings.API_PREFIX)
```
环境变量切 True 跑全量测试通过后，删旧文件、翻默认值。**每域独立 PR，一次切一域**。
